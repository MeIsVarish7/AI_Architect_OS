# Dynamic Prompt Generator

## Purpose

The Dynamic Prompt Generator is the final stage of AI Architect OS.

Its responsibility is to convert the current operating system state into a single Dynamic Session Prompt that can be understood by any capable AI.

The Operating System provides context.

The AI provides intelligence.

The Dynamic Prompt Generator connects both.

---

## Primary Mission

Generate a context-aware prompt that allows any AI to immediately continue my learning journey without unnecessary setup or repeated questioning.

---

## Inputs

Before generating a prompt, gather information from:

- AI Manager
- Context Engine
- Session Workflow
- State Management
- Prompt Templates

Along with:

- Kernel
- Curriculum
- Daily Context
- Weekly Planner
- Monthly Planner
- Projects Dashboard
- Research Dashboard
- GitHub Dashboard
- BlackBook
- Reviews Dashboard
- Resources Dashboard

---

## Prompt Generation Process

Every generated prompt follows this sequence:

1. Read Operating System
2. Reconstruct Current State
3. Determine Current Objective
4. Select Appropriate Prompt Template
5. Populate Template Variables
6. Generate Dynamic Session Prompt

---

## Prompt Components

Every generated prompt should include:

- Current learning state
- Current curriculum position
- Current topic
- Previous session summary
- Available study time
- Active projects
- Research stage
- Current priorities
- Teaching strategy
- Verification strategy
- Expected session outcome

Only include information relevant to the current session.

---

## Prompt Principles

Every prompt should:

- Continue from the previous session.
- Preserve learning continuity.
- Respect curriculum dependencies.
- Adapt to available time.
- Adapt to demonstrated competency.
- Avoid unnecessary repetition.
- Minimize unnecessary questions.

---

## Prompt Output

The generated prompt should transform any compatible AI into my Technical Learning Manager for the current session.

The prompt should contain everything required to begin learning immediately.

---

## Session Completion

At the end of every learning session the AI generates a Session Update.

The Session Update becomes part of AI Architect OS and is used during the next prompt generation cycle.

---

## Constraints

The Dynamic Prompt Generator should never:

- Restart completed topics.
- Ignore previous progress.
- Skip prerequisite knowledge.
- Generate duplicate work.
- Lose learning continuity.
- Depend on one specific AI model.

---

## Success Criteria

A successful Dynamic Prompt Generator should allow any capable AI to:

- Instantly understand my current state.
- Continue from my last completed work.
- Teach adaptively.
- Mentor throughout the session.
- Generate a Session Update.
