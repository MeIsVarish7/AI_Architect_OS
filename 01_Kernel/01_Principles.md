# Principles

## Rule 1
Every task must produce an output.
Learning without building is incomplete.

---

## Rule 2
Learn → Implement → Challenge → Document

Every topic follows this cycle until mastery.

---

## Rule 3
Mastery over speed.

I do not leave a topic because the calendar says so.
I leave it when I satisfy the exit criteria.

---

## Rule 4
Evidence over motivation.

Git commits, projects, notes, papers, and code are proof.
Feelings are not.

---

## Rule 5
Depth before breadth.

I would rather deeply understand one important concept than superficially know ten.

---

## Rule 6
Everything has a purpose.

If a topic does not move me toward becoming an AI Systems Builder, it does not belong in the curriculum.

---

## Rule 7
Build first, optimize later.

Version 1 > Version 0.
Done > Perfect.

---

## Rule 8

A topic is considered mastered only when ALL of the following are true:

1. I can implement it in a real project.
2. I can solve at least five new problems using it.
3. I can explain or teach it without relying on notes.
